The unified `deploy.sh` script provides an interactive, automated deployment process for both PDF accessibility solutions. This is the **recommended deployment method** for most users.

## Overview

The deployment script handles:

- Solution selection (PDF-to-PDF or PDF-to-HTML)
- AWS credential verification
- Solution-specific configuration (Adobe credentials or Bedrock setup)
- IAM role creation with scoped permissions
- CodeBuild project creation and execution
- Real-time deployment monitoring
- Optional frontend UI deployment

<Tip>
  The script supports cascading deployments - you can deploy one solution first, then add the other, and optionally deploy the UI for both.
</Tip>

---

## Before You Begin

Ensure you've completed all [prerequisites](/deployment/prerequisites):

- AWS account with appropriate permissions
- CloudShell access or AWS CLI configured
- Adobe credentials (for PDF-to-PDF) or Bedrock Data Automation access (for PDF-to-HTML)

## Deployment Process

### Step 1: Clone the Repository

Open AWS CloudShell and clone the repository:

```bash
git clone https://github.com/ASUCICREPO/PDF_Accessibility.git
cd PDF_Accessibility
```

<Note>
  The repository contains all necessary infrastructure code, Lambda functions, and Docker configurations.
</Note>

### Step 2: Run the Deployment Script

Make the script executable and run it:

```bash
chmod +x deploy.sh
./deploy.sh
```

The script will display a welcome screen with solution options:

```
🎉 Welcome to PDF Accessibility Solutions Enhanced Deployment! 🎉
=====================================================================

This tool will help you deploy PDF accessibility solutions:

1. 📄 PDF-to-PDF Remediation
   • Maintains original PDF format
   • Uses Adobe PDF Services API
   • Advanced processing with ECS and Step Functions
   • Best for: Documents that must remain as PDFs

2. 🌐 PDF-to-HTML Remediation
   • Converts PDFs to accessible HTML
   • Uses AWS Bedrock Data Automation
   • Serverless Lambda-based processing
   • Best for: Web-accessible content

3. 🎨 Frontend UI
   • Web interface for both solutions
   • User authentication with Cognito
   • File upload and processing monitoring
   • Best for: End-user accessibility
```

### Step 3: Select Your Solution

The script will prompt you to choose your first solution:

```
Which solution would you like to deploy first?
1) PDF-to-PDF Remediation
2) PDF-to-HTML Remediation

Enter your choice (1 or 2):
```

<Tabs>
  <Tab title="PDF-to-PDF">
    Choose **option 1** if you need to:
    - Maintain PDF format for regulatory compliance
    - Process documents that must remain as PDFs
    - Use Adobe's enterprise-grade PDF processing
    - Handle complex PDF structures (forms, annotations)
  </Tab>
  <Tab title="PDF-to-HTML">
    Choose **option 2** if you need to:
    - Convert documents to web-accessible HTML
    - Optimize for screen readers and assistive technology
    - Reduce file sizes and improve load times
    - Serve content directly on websites
  </Tab>
</Tabs>

## PDF-to-PDF Deployment

### Adobe Credentials Setup

After selecting PDF-to-PDF, the script will prompt for Adobe API credentials:

```bash
Adobe PDF Services API credentials are required:
(These will be stored securely in AWS Secrets Manager)
   Enter Adobe API Client ID: your-client-id-here
   ✅ Client ID received!
   Enter Adobe API Client Secret: your-client-secret-here
   ✅ Client Secret received!
```

<Warning>
  Never share or commit your Adobe credentials. The script stores them securely in AWS Secrets Manager.
</Warning>

The script will:

1. Create a JSON credentials file
2. Upload to AWS Secrets Manager at `/myapp/client_credentials`
3. Configure ECS tasks and Lambda functions to access the secret
4. Delete the temporary credentials file

## Cascading Deployment Options

After deploying your first solution, the script offers additional options:

```bash
🤔 What would you like to do next?
==================================

1) Deploy PDF-to-HTML Remediation (complete both backend solutions)
2) Deploy Frontend UI (for current solution)
3) Finish (backend only)

Enter your choice (1, 2, or 3):
```

### Deploy Both Solutions

Choose **option 1** to deploy the other solution:

- Creates a new CodeBuild project for the second solution
- Configures solution-specific IAM permissions
- Deploys infrastructure independently
- Both solutions can be used simultaneously

After deploying both backends, you'll be prompted to deploy the UI:

```bash
🎉 Both backend solutions deployed successfully!

Would you like to deploy the Frontend UI? (y/n):
```

### Deploy Frontend UI Only

Choose **option 2** to deploy the web interface for your current solution(s). See [UI Deployment](/deployment/ui-deployment) for details.

### Finish Deployment

Choose **option 3** to complete the deployment without additional components.

## Troubleshooting Deployment Issues

### Build Failures

<AccordionGroup>
  <Accordion title="CodeBuild Permission Errors">
    **Symptom:**
    ```
    AccessDenied: User is not authorized to perform: iam:CreateRole
    ```
    
    **Solution:**
    - Verify your AWS user has the required IAM permissions
    - See [IAM Permissions Guide](/configuration/iam-permissions)
    - Contact your AWS administrator to grant necessary permissions
  </Accordion>

  <Accordion title="ECR Login Failures">
    **Symptom:**
    ```
    ERROR: ECR login failed!
    ```
    
    **Solution:**
    - Verify ECR permissions in your IAM policy
    - Check that the region is correctly set
    - Ensure Docker is running (if using local AWS CLI)
  </Accordion>

  <Accordion title="Docker Build Failures">
    **Symptom:**
    ```
    ERROR: Docker build failed!
    ```
    
    **Solution:**
    - Check CodeBuild logs for specific error messages
    - Verify the Dockerfile syntax
    - Ensure base images are accessible
    - Try increasing CodeBuild compute size to LARGE
  </Accordion>

  <Accordion title="Elastic IP Quota Exceeded">
    **Symptom:**
    ```
    Resource handler returned message: "The maximum number of addresses has been reached."
    ```
    
    **Solution:**
    1. Go to [EC2 Service Quotas](https://console.aws.amazon.com/servicequotas/home/services/ec2/quotas)
    2. Search for "Elastic IPs"
    3. Request quota increase
    4. Re-run deployment after approval (usually within minutes)
  </Accordion>

  <Accordion title="CDK Bootstrap Failures">
    **Symptom:**
    ```
    CDK bootstrap failed
    ```
    
    **Solution:**
    - Verify CloudFormation permissions
    - Check S3 permissions for CDK staging buckets
    - Ensure the region supports CDK
    - Try manually bootstrapping:
      ```bash
      cdk bootstrap aws://ACCOUNT-ID/REGION
      ```
  </Accordion>
</AccordionGroup>

### Viewing Build Logs

To troubleshoot failed builds:

```bash
# List recent build runs
aws codebuild list-builds-for-project --project-name YOUR-PROJECT-NAME

# Get build details
aws codebuild batch-get-builds --ids BUILD-ID

# View CloudWatch logs
LOG_GROUP="/aws/codebuild/YOUR-PROJECT-NAME"
aws logs tail $LOG_GROUP --follow
```

Or view in the AWS Console:
1. Navigate to [CodeBuild Console](https://console.aws.amazon.com/codesuite/codebuild/projects)
2. Click your project name
3. View build history and logs

## Redeployment

To redeploy or update an existing deployment:

### Using the Script

Simply re-run the deployment script:

```bash
cd PDF_Accessibility
./deploy.sh
```

The script will:
- Reuse existing IAM roles if found
- Update existing secrets if changed
- Create a new CodeBuild project with a fresh timestamp
- Deploy updated infrastructure

### Using CodeBuild Directly

Trigger a build manually:

```bash
aws codebuild start-build \
  --project-name pdfremediation-TIMESTAMP \
  --source-version main
```

### Using AWS Console

1. Navigate to [CodeBuild Console](https://console.aws.amazon.com/codesuite/codebuild/projects)
2. Select your project
3. Click **Start build**
4. Choose branch (default: `main`)
5. Click **Start build**

<Tip>
  Keep your CodeBuild project names for easy redeployment. You can find them in the console or by running:
  ```bash
  aws codebuild list-projects
  ```
</Tip>
