# Quickstart Guide

Get PDF Accessibility Solutions up and running in your AWS account with our automated one-click deployment script. This guide walks you through the fastest path to a working deployment.

## Prerequisites

Before you begin, ensure you have:

<CardGroup cols={2}>
  <Card title="AWS Account" icon="aws">
    Active AWS account with appropriate permissions
  </Card>
  <Card title="CloudShell Access" icon="terminal">
    Ability to launch AWS CloudShell from the management console
  </Card>
  <Card title="Adobe API Credentials" icon="key">
    Required for PDF-to-PDF solution (trial or enterprise account)
  </Card>
  <Card title="Bedrock Access" icon="brain">
    AWS Bedrock enabled in your region for AI-powered features
  </Card>
</CardGroup>

<Note>
See [Prerequisites](/deployment/prerequisites) for detailed requirements and IAM permission setup.
</Note>

## One-Click Deployment

<Steps>
  <Step title="Open AWS CloudShell">
    1. Sign in to the AWS Management Console
    2. Click the **CloudShell** icon in the top navigation bar
    3. Wait for CloudShell to initialize (may take a moment on first use)
  </Step>

  <Step title="Clone the Repository">
    ```bash
    git clone https://github.com/ASUCICREPO/PDF_Accessibility.git
    cd PDF_Accessibility
    ```
  </Step>

  <Step title="Run the Deployment Script">
    ```bash
    chmod +x deploy.sh
    ./deploy.sh
    ```

    The script will guide you through:
    - **Solution selection** (PDF-to-PDF, PDF-to-HTML, or both)
    - **Credential setup** (Adobe API for PDF-to-PDF)
    - **Automated deployment** via AWS CodeBuild
    - **Optional UI deployment** for end users
  </Step>

  <Step title="Configure Your Solution">
    <Tabs>
      <Tab title="PDF-to-PDF">
        When prompted, enter your Adobe PDF Services API credentials:

        ```bash
        Enter Adobe API Client ID: your-client-id-here
        Enter Adobe API Client Secret: your-secret-here
        ```

        The script stores these securely in AWS Secrets Manager at `/myapp/client_credentials`.
      </Tab>

      <Tab title="PDF-to-HTML">
        No manual configuration needed! The script automatically:
        - Creates a Bedrock Data Automation project
        - Sets up the S3 bucket structure
        - Builds and deploys the Lambda container
      </Tab>
    </Tabs>
  </Step>

  <Step title="Monitor Deployment">
    Watch the deployment progress in real-time:

    ```log
    [Install] Installing dependencies...
    [Pre_Build] Configuring AWS CDK...
    [Build] Deploying CloudFormation stack...
    [Post_Build] Deployment complete!
    ```

    **Deployment time:**
    - PDF-to-PDF: 3-5 minutes
    - PDF-to-HTML: 5-10 minutes
  </Step>
</Steps>

## Test Your Deployment

<Tabs>
  <Tab title="PDF-to-PDF">
    ### 1. Find Your S3 Bucket

    ```bash
    aws s3 ls | grep pdfaccessibility
    ```

    Bucket name format: `pdfaccessibility-<unique-id>`

    ### 2. Upload a Test PDF

    ```bash
    aws s3 cp sample.pdf s3://pdfaccessibility-<unique-id>/pdf/sample.pdf
    ```

    Or use the AWS Console:
    - Navigate to your S3 bucket
    - Upload files to the `pdf/` folder (created automatically on first upload)

    ### 3. Monitor Processing

    Processing happens automatically:
    - **Temporary files**: Watch the `temp/` folder for intermediate chunks
    - **Final results**: Check the `result/` folder for `COMPLIANT_sample.pdf`

    View the CloudWatch dashboard for real-time progress:

    ```bash
    aws cloudwatch list-dashboards | grep PDF_Processing
    ```

    ### 4. Download Results

    ```bash
    aws s3 cp s3://pdfaccessibility-<unique-id>/result/COMPLIANT_sample.pdf .
    ```
  </Tab>

  <Tab title="PDF-to-HTML">
    ### 1. Find Your S3 Bucket

    ```bash
    aws s3 ls | grep pdf2html-bucket
    ```

    Bucket name format: `pdf2html-bucket-<account-id>-<region>`

    ### 2. Upload a Test PDF

    ```bash
    aws s3 cp sample.pdf s3://pdf2html-bucket-<account-id>-<region>/uploads/sample.pdf
    ```

    The `uploads/` folder is created automatically during deployment.

    ### 3. Monitor Processing

    Check CloudWatch logs:

    ```bash
    aws logs tail /aws/lambda/Pdf2HtmlPipeline --follow
    ```

    ### 4. Download Results

    ```bash
    # List remediated files
    aws s3 ls s3://pdf2html-bucket-<account-id>-<region>/remediated/

    # Download the final package
    aws s3 cp s3://pdf2html-bucket-<account-id>-<region>/remediated/final_sample.zip .

    # Extract contents
    unzip final_sample.zip
    ```

    The ZIP contains:
    - `remediated.html` - Final accessibility-compliant HTML
    - `result.html` - Original HTML conversion
    - `images/` - Extracted images with AI-generated alt text
    - `remediation_report.html` - Detailed accessibility improvements report
    - `usage_data.json` - Processing metrics
  </Tab>
</Tabs>

## Verify Accessibility

<Accordion title="How to verify WCAG 2.1 Level AA compliance">
  Use these tools to validate your remediated documents:

  **For PDFs:**
  - Adobe Acrobat Pro (Accessibility Checker)
  - PAC 2024 (free tool from PDF/UA Foundation)
  - PAVE (PDF Accessibility Validation Engine)

  **For HTML:**
  - WAVE (Web Accessibility Evaluation Tool)
  - axe DevTools browser extension
  - Lighthouse accessibility audit in Chrome DevTools

  Look for:
  - ✅ Proper heading hierarchy
  - ✅ Alt text on all images
  - ✅ Semantic structure and tags
  - ✅ Language identification
  - ✅ Color contrast compliance
  - ✅ Table markup
  - ✅ Form labels
</Accordion>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Deployment fails with permission errors">
    **Solution:** Verify your AWS credentials have the required IAM permissions. See [IAM Permissions](/configuration/iam-permissions) for the complete list.
  </Accordion>

  <Accordion title="Adobe API authentication fails">
    **Solution:** 
    - Verify your Adobe credentials are correct
    - Check that your Adobe account is active (trial or enterprise)
    - Ensure credentials are properly stored in Secrets Manager

    ```bash
    aws secretsmanager get-secret-value --secret-id /myapp/client_credentials
    ```
  </Accordion>

  <Accordion title="Bedrock model access denied">
    **Solution:** Enable Bedrock model access in the AWS Console:
    1. Navigate to Amazon Bedrock
    2. Go to **Model access**
    3. Enable **Amazon Nova Pro** and **Amazon Nova Lite**
    
    See [Bedrock Setup](/configuration/bedrock-setup) for detailed instructions.
  </Accordion>

  <Accordion title="Processing takes too long or times out">
    **Solution:** 
    - Check CloudWatch logs for specific errors
    - Verify your document isn't exceeding size limits
    - See [Limits and Quotas](/configuration/limits-and-quotas) for configuration options
  </Accordion>
</AccordionGroup>
